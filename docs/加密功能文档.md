本文档描述了项目中的所有加密相关功能，包括国密（SM2/SM3/SM4）算法支持和标准TLS加密套件。

## 三、国密SSL/TLS上下文 (GmSslContext)

### 3.1 功能说明

`GmSslContext` 是国密SSL/TLS上下文工厂类，用于构建支持国密算法的服务端和客户端SSL上下文。

### 3.2 支持的加密套件

#### 国密加密套件

| 加密套件名称 | 密钥交换 | 签名算法 | 加密算法 | 哈希算法 |
|------------|---------|---------|---------|---------|
| `TLS_ECDHE_ECDSA_WITH_SM4_SM3` | ECDHE | SM2/ECDSA | SM4 | SM3 |
| `TLS_ECDHE_RSA_WITH_SM4_SM3` | ECDHE | SM2/RSA | SM4 | SM3 |
| `TLS_ECDH_ECDSA_WITH_SM4_SM3` | ECDH | SM2/ECDSA | SM4 | SM3 |
| `TLS_ECDH_RSA_WITH_SM4_SM3` | ECDH | SM2/RSA | SM4 | SM3 |
| `ECDHE_SM4_SM3` | ECDHE | SM2 | SM4 | SM3 |
| `ECC_SM4_SM3` | ECC | SM2 | SM4 | SM3 |

#### 标准TLS备选套件

| 加密套件 | 密钥交换 | 签名算法 | 加密算法 |
|---------|---------|---------|---------|
| `TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256` | ECDHE | ECDSA | AES-128-GCM |
| `TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256` | ECDHE | RSA | AES-128-GCM |
| `TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384` | ECDHE | ECDSA | AES-256-GCM |
| `TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384` | ECDHE | RSA | AES-256-GCM |
| `TLS_RSA_WITH_AES_128_GCM_SHA256` | RSA | RSA | AES-128-GCM |
| `TLS_RSA_WITH_AES_256_GCM_SHA384` | RSA | RSA | AES-256-GCM |

### 3.3 使用方法

```java
// 服务端构建
SSLContext sslContext = GmSslContext.forServer()
    .keyManager(credentialManager)
    .trustManager(trustManager)
    .clientAuth(ClientAuth.REQUIRED)
    .build();

// 客户端构建
SSLContext sslContext = GmSslContext.forClient()
    .keyManager(credentialManager)
    .trustManager(trustManager)
    .build();

// 获取SSL上下文
SSLContext context = sslContext.getSslContext();
```

### 3.4 Provider配置

- 国密TLS: `BCJSSE` (BouncyCastle JSSE Provider)
- 标准TLS: `BC` (BouncyCastle Provider)
- KeyStore格式: `PKCS12`
- 协议版本: `TLSv1.2`

## 四、SM2签名与SM3哈希 (GmSignature)

### 4.1 算法配置

```java
private static final String SIGNATURE_ALGORITHM = "SM3withSM2";
private static final String KEY_ALGORITHM = "SM2";
private static final String PROVIDER = "BC";
```

### 4.2 椭圆曲线参数

- **曲线名称**: `sm2p256v1`
- **密钥长度**: 256位
- **签名长度**: 64字节 (r || s 格式)

### 4.3 API文档

#### 签名方法

```java
/**
 * SM2签名
 * @param privateKey SM2私钥
 * @param data 待签名数据
 * @return 64字节签名值
 */
byte[] sign(PrivateKey privateKey, byte[] data)
```

#### 验签方法

```java
/**
 * SM2验签
 * @param publicKey SM2公钥
 * @param data 原始数据
 * @param signature 签名值
 * @return 验签结果
 */
boolean verify(PublicKey publicKey, byte[] data, byte[] signature)
```

#### SM3哈希

```java
/**
 * SM3哈希
 * @param data 待哈希数据
 * @return 32字节哈希值
 */
byte[] sm3(byte[] data)

/**
 * SM3哈希(返回十六进制字符串)
 * @param data 待哈希数据
 * @return 64字符十六进制字符串
 */
String sm3Hex(String data)
String sm3Hex(byte[] data)
```

#### 密钥编解码

```java
/**
 * 从十六进制字符串解码公钥
 */
PublicKey decodePublicKey(String hex)

/**
 * 从字节数组解码公钥
 */
PublicKey decodePublicKey(byte[] encoded)

/**
 * 从十六进制字符串解码私钥
 */
PrivateKey decodePrivateKey(String hex)

/**
 * 从字节数组解码私钥
 */
PrivateKey decodePrivateKey(byte[] encoded)

/**
 * 生成SM2密钥对
 */
KeyPair generateKeyPair()
```

### 4.4 使用示例

```java
// 生成SM2密钥对
KeyPair keyPair = GmSignature.generateKeyPair();

// SM2签名
byte[] signature = GmSignature.sign(keyPair.getPrivate(), data);

// SM2验签
boolean valid = GmSignature.verify(keyPair.getPublic(), data, signature);

// SM3哈希
byte[] hash = GmSignature.sm3(data);
String hexHash = GmSignature.sm3Hex(data);
```

## 五、凭证管理 (GmCredentialManager)

### 5.1 功能说明

`GmCredentialManager` 负责从PKCS12密钥库加载和管理国密凭证。

### 5.2 创建方式

```java
// 从PKCS12密钥库加载服务端凭证
GmCredentialManager server = GmCredentialManager.forServer(
    "keystore.p12",    // 密钥库路径
    "password",        // 密钥库密码
    "alias"            // 证书别名
);

// 从PKCS12密钥库加载客户端凭证
GmCredentialManager client = GmCredentialManager.forClient(
    "client.p12",
    "password",
    "alias"
);

// 从证书文件创建只读凭证
GmCredentialManager fromCert = GmCredentialManager.fromCertificate("cert.pem");

// 从密钥和证书创建
GmCredentialManager fromPair = GmCredentialManager.fromKeyAndCert(
    privateKey,
    certificate
);
```

### 5.3 辅助方法

```java
// 验证证书有效性
void validateCertificate(Date validateDate)

// 获取证书十六进制编码
String getCertificateHex()

// 获取证书Base64编码
String getCertificateBase64()

// 获取证书SHA-256指纹
String getCertificateFingerprint()
```

## 七、认证验证器 (GmAuthenticator)

### 7.1 安全特性

- **时间戳容差**: 默认300秒，用于防止重放攻击

### 7.2 创建方式

```java
// 使用信任证书创建
GmAuthenticator auth = new GmAuthenticator(trustedCert);

// 带自定义时间容差创建
GmAuthenticator auth = new GmAuthenticator(trustedCert, 600L);

// 使用信任管理器创建
GmAuthenticator auth = new GmAuthenticator(trustManager);
```

### 7.3 验证方法

```java
// 完整验证（包括时间戳）
Optional<CmsServiceError> validate(
    AuthenticationParameter param,    // 认证参数
    byte[] signature                  // 签名值
)

// 简化验签（仅验证签名）
Optional<CmsServiceError> validateSimple(
    AuthenticationParameter param,
    PublicKey publicKey,
    byte[] signature
)
```

## 八、信任管理器 (GmTrustManager)

### 8.1 创建方式


### 8.2 添加可信证书

```java

// 检查指纹是否可信
```

| 类型 | 算法 | 密钥长度 | 有效期 |

| CA根证书 | RSA | 2048位 | 365天 |
| 服务端证书 | RSA | 2048位 | 365天 |

### 9.3 生成的文件

- `certs/ca.p12` - CA密钥库

- `certs/client.p12` - 客户端密钥库
- `certs/ca.pem` - CA证书(PEM)
- `certs/server.pem` - 服务端证书(PEM)
- `certs/client.pem` - 客户端证书(PEM)

> **注意**: 测试脚本使用RSA算法生成证书。生产环境应使用GmSSL工具生成国密证书。

## 十一、算法汇总

| 算法类型 | 算法名称 | 密钥长度 | 用途 |
|---------|---------|---------|------|
| **签名算法** | SM3withSM2 | 256位 | 数字签名 |
| **哈希算法** | SM3 | 256位 | 消息摘要 |
| **对称加密** | SM4 | 128位 | 数据加密 |
| **椭圆曲线** | sm2p256v1 | 256位 | 国密密钥交换/签名 |
| **椭圆曲线** | secp256r1 | 256位 | 标准TLS备用 |
| **证书格式** | X.509 | - | 证书存储 |
| **密钥库** | PKCS12 | - | 密钥存储 |

## 十二、标准遵循

- **GB/T 32918.4-2016**: 信息安全技术 SM2椭圆曲线公钥密码算法
- **DL/T 2811-2024**: 变电站通信网络和系统 - CMS协议实现
- **TLS 1.2**: 传输层安全协议

---

## 十三、注意事项

1. **国密证书**: 生产环境应使用GmSSL工具生成符合国密标准的证书
2. **密钥安全**: 妥善保管私钥和密钥库密码
3. **证书有效期**: 定期更新证书，避免过期
4. **TLS版本**: 当前仅支持TLS 1.2，不支持TLS 1.3
5. **双向认证**: 高安全场景应启用客户端证书认证
