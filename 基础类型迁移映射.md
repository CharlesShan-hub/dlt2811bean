# IEC 61850 → DL/T 2811 基础数据类型迁移映射

> 基于对 iec61850bean (com.beanit.iec61850bean) 全量源码分析和 dlt2811bean (com.ysh.dlt2811bean) 现有结构的对比。
>
> **核心结论**：基础数据类型的语义定义完全一致，差异仅在编码层（MMS-BER → CMS-PER）。

---

## 二、逐类型详细映射表

### 2.1 整数类型

| 标准类型 | iec61850bean 类 | 文件路径 | Java 内部存储 | 迁移到 dlt2811bean 的目标位置 |
|---------|----------------|----------|-------------|---------------------------|
| INT8 | `BdaInt8` | `src/main/java/.../BdaInt8.java` | `byte` (有符号) | `core/model/datatype/BdaInt8.java` |
| INT16 | `BdaInt16` | 同目录 | `short` | `core/model/datatype/BdaInt16.java` |
| INT32 | `BdaInt32` | 同目录 | `int` | `core/model/datatype/BdaInt32.java` |
| INT64 | `BdaInt64` | 同目录 | `long` | `core/model/datatype/BdaInt64.java` |
| INT128 | `BdaInt128` | 同目录 | `long` | `core/model/datatype/BdaInt128.java` |
| INT8U | `BdaInt8U` | 同目录 | `short` (Java 无 byte 无符号) | `core/model/datatype/BdaInt8U.java` |
| INT16U | `BdaInt16U` | 同目录 | `int` | `core/model/datatype/BdaInt16U.java` |
| INT32U | `BdaInt32U` | 同目录 | `long` | `core/model/datatype/BdaInt32U.java` |

**迁移要点**：
- 值存储逻辑（getValue/setValue/setDefault/copy）→ **完全复用**
- 编码方法 `getMmsDataObj()` / `setValueFromMmsDataObj()` / `getMmsTypeSpec()` → **删除或替换为 PER 编码**
- 构造函数签名保持不变：`(ObjectReference, Fc, sAddr, dchg, dupd)`

### 2.2 浮点类型

| 标准类型 | iec61850bean 类 | 存储 | 特殊说明 | 目标位置 |
|---------|----------------|------|---------|---------|
| FLOAT32 | `BdaFloat32` | `byte[5]` | 前 1 字节=指数宽度(8)，后 4 字节=IEEE754 float | `core/model/datatype/BdaFloat32.java` |
| FLOAT64 | `BdaFloat64` | `byte[9]` | 前 1 字节=指数宽度(11)，后 8 字节=IEEE754 double | `core/model/datatype/BdaFloat64.java` |

**迁移要点**：
- `getFloat()/setFloat(Float)` / `getDouble()/setDouble(Double)` 方法 → **完全复用**
- 默认值格式 `{expWidth, 0,0,...}` → **复用**

### 2.3 布尔与字符串类型

| 标准类型 | iec61850bean 类 | 说明 | 目标位置 |
|---------|----------------|------|---------|
| BOOLEAN | `BdaBoolean` | 内部用 `boolean` | `core/model/datatype/BdaBoolean.java` |
| VISIBLE_STRING | `BdaVisibleString` | 带 maxLength 参数，UTF-8 字节存储 | `core/model/datatype/BdaVisibleString.java` |
| UNICODE_STRING | `BdaUnicodeString` | 带 maxLength 参数 | `core/model/datatype/BdaUnicodeString.java` |
| OCTET_STRING | `BdaOctetString` | 带 maxLength 参数 | `core/model/datatype/BdaOctetString.java` |

### 2.4 位字符串类型（继承自 BdaBitString）

| 标准类型 | iec61850bean 类 | bit 数 | 说明 | 目标位置 |
|---------|----------------|--------|------|---------|
| Quality | `BdaQuality` | 13 bit | 含 Validity 枚举(GOOD/INVALID/RESERVED/QUESTIONABLE) | `core/model/datatype/BdaQuality.java` |
| Dbpos | `BdaDoubleBitPos` | 2 bit | DoubleBitPos 枚举(INTERMEDIATE/OFF/ON/BAD) | `core/model/datatype/BdaDoubleBitPos.java` |
| Check | `BdaCheck` | 2 bit | synchrocheck(第7位) + interlockCheck(第6位) | `core/model/datatype/BdaCheck.java` |
| Tcmd | `BdaTapCommand` | 2 bit | TapCommand 枚举(STOP/LOWER/HIGHER/RESERVED) | `core/model/datatype/BdaTapCommand.java` |
| TriggerOptions | `BdaTriggerConditions` | 6 bit | dchg/qchg/dupd/integrity/GI | `core/model/datatype/BdaTriggerConditions.java` |
| OptFields | `BdaOptFlds` | 10 bit | seqNum/rptTS/reasonForInclusion/dataRef/bufOverflow/entryId/confRev/seg | `core/model/datatype/BdaOptFlds.java` |
| - | `BdaReasonForInclusion` | 7 bit | dchg/qchg/dupd/integrity/GI/appTrigger | `core/model/datatype/BdaReasonForInclusion.java` |

**注意**：`BdaBitString` 本身也是需要迁移的抽象基类。

### 2.5 时间类型

| 标准类型 | iec61850bean 类 | 字节数 | 说明 | 目标位置 |
|---------|----------------|--------|------|---------|
| Timestamp / UtcTime | `BdaTimestamp` | 8 字节 | SecondsSinceEpoch(4) + FractionOfSecond(3) + TimeQuality(1) | `core/model/datatype/BdaTimestamp.java` |
| EntryTime / BinaryTime | `BdaEntryTime` | 6 字节 | TimeOfDay(4ms) + DaySince1984(2) | `core/model/datatype/BdaEntryTime.java` |

### 2.6 引用与辅助类型

| 标准类型 | iec61850bean 类 | 说明 | 目标位置 |
|---------|----------------|------|---------|
| ObjectReference | `ObjectReference` | LD/LN/DO/DA 引用解析器 | `core/model/ObjectReference.java` |
| FunctionalConstraint | `Fc` (enum) | ST/MX/SP/CF/DC/SG/SE/SR/OR/BL/EX/CO/RP/BR | `core/model/Fc.java` |
| BdaType | `BdaType` (enum) | 类型枚举标识 | `core/model/BdaType.java` |
| ServiceError | `ServiceError` | 错误码常量 + Exception 类 | `core/ServiceError.java` |
| FileEntry | `FileInformation` | {fileName, fileSize, lastModified} | `core/model/FileInformation.java` |

### 2.7 控制模型相关（无独立 Bda 类）

这些在 iec61850bean 中**没有独立的 Java 类**，而是通过 SCL 解析时动态组合：

| 标准类型 | iec61850bean 中的实现方式 | 说明 | 迁移方案 |
|---------|------------------------|------|---------|
| Originator (orCat+orIdent) | 在 SCL ICD 中定义为 DAType，运行时 orCat=BdaInt8, orIdent=BdaOctetString | ClientAssociation 注释中提到 | 需新建 `BdaOriginator.java` 或在控制服务中处理 |
| AddCause | ❌ 未找到独立类 | 61850-7-2 定义但 iec61850bean 未单独建模 | 需新建 `AddCause` 枚举/类（DL/T 2811 扩展至 27 个值） |
| PHYCOMADDR | ⚠️ SclParser 中用 `BdaOctetString(..., 6, ...)` 临时替代 | TODO 注释说 "not correct!" | 需新建 `BdaPhyComAddr.java`（含 addr+priority+vid+appid） |

## 三、dlt2811bean 当前已有的 PER 编码基础

```
com.ysh.dlt281bean.core.per/              # ✅ 已有 PER 编码框架

├── CmsPerType.java                        # CMS PER 类型基础接口/枚举

    ├── PerReal.java                       # 浮点数 PER 编码
    ├── PerBitString.java                  # 位串 PER 编码
    ├── PerBoolean.java                    # 布尔值 PER 编码
    ├── PerEnumerated.java                 # 枚举 PER 编码
    ├── PerOctetString.java                # 八位字节串 PER 编码
    ├── PerVisibleString.java              # 可见字符串 PER 编码

```

**关键发现**：dlt2811bean 已经有了完整的 PER 编码原语！这意味着：

> **你不需要写 PER 编码逻辑，只需要把 iec61850bean 的 Bda* 类中的 MMS 编码替换为调用 per.types.* 即可。**

### 每个类的改造模板

以 `BdaInt8` 为例，改动只有 3 处：

```diff
- package com.beanit.iec61850bean;
+ package com.ysh.dlt281bean.core.model.datatype;

- import com.beanit.asn1bean.ber.types.BerInteger;
- import com.beanit.iec61850bean.internal.mms.asn1.Data;
- import com.beanit.iec61850bean.internal.mms.asn1.TypeDescription;
- import com.beanit.iec61850bean.internal.mms.asn1.Unsigned8;
+ import com.ysh.dlt281bean.core.per.types.PerInteger;

  public final class BdaInt8 extends BasicDataAttribute {
    private volatile byte value;
    // ... 构造函数、setValue、getValue、setDefault、copy、toString 全部不变 ...

-   @Override
-   Data getMmsDataObj() { ... }          // ❌ 删除此方法
-
-   @Override
-   void setValueFromMmsDataObj(Data data) throws ServiceError { ... }  // ❌ 删除此方法
-
-   @Override
-   TypeDescription getMmsTypeSpec() { ... }  // ❌ 删除此方法
+
+   // ✅ 新增 PER 编码方法
+   public void encodePer(PerOutputStream out) {
+     PerInteger.encode(out, value, (byte)-128, (byte)127);
+   }
+
+   public void decodePer(PerInputStream in) throws PerDecodeException {
+     value = (byte) PerInteger.decode(in, (byte)-128, (byte)127);
+   }
  }
```

| 类别 | 数量 | 每个类改动比例 | 说明 |
|------|------|------------|------|

## 五、执行顺序建议

### 第一批（最核心，优先做）：框架 + 整数 + 浮点 + 布尔
1. `ModelNode` → `FcModelNode` → `BasicDataAttribute` （三个基类，建立继承体系）
2. `Fc` 枚举 + `BdaType` 枚举 + `ObjectReference`
3. `BdaInt8/16/32/64/128/U8/U16/U32`
4. `BdaFloat32/64`
5. `BdaBoolean`

### 第二批（字符串 + 时间 + 位串基础）
6. `BdaBitString`（抽象基类）
7. `BdaVisibleString/UnicodeString/OctetString`
8. `BdaTimestamp/EntryTime`

### 第三批（具体位字符串）
9. `BdaQuality`（最重要，几乎所有数据属性都有）
10. `BdaDoubleBitPos/Check/TapCommand`
11. `BdaTriggerConditions/OptFlds/ReasonForInclusion`

### 第四批（辅助 + 新增）
12. `ServiceError` + `FileInformation`
13. `BdaOriginator` + `BdaPhyComAddr` + `AddCause`

## 六、注意事项

1. **License**：iec61850bean 使用 Apache License 2.0，可以自由使用和修改，只需保留版权声明。

2. **包名替换**：全局替换 `com.beanit.iec61850bean` → `com.ysh.dlt281bean.core.model`

3. **MMS 依赖全部清除**：所有 `import com.beanit...internal.mms.*` 和 `import com.beanit.asn1bean.ber.*` 都要删掉

4. **BdaQuality.Validity 枚举**是内嵌枚举，保留它——DL/T 2811 完全一样

5. **BdaDoubleBitPos.DoubleBitPos 枚举**同上

6. **BdaTapCommand.TapCommand 枚举**同上

7. **BdaCheck** 固定使用 `Fc.CO`（控制操作），这个在 dlt2811bean 中可能需要调整

8. **默认值必须一致**：每个类的 `setDefault()` 方法定义的默认值是标准规定的，不要改
